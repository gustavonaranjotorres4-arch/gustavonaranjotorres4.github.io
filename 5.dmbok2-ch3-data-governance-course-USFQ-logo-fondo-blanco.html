¿Qué es Data Governance?

El Capítulo 3 del DMBOK2 define Data Governance (DG) como el ejercicio de autoridad, control y toma de decisiones compartidas sobre la gestión de activos de datos. Es el eje central que conecta todos los Knowledge Areas del DAMA Wheel.

Definición DMBOK2

Data Governance es el ejercicio de autoridad, control y toma de decisiones compartidas (planificación, monitoreo y aplicación) sobre la gestión de activos de datos. Incluye las personas, los procesos y las tecnologías requeridas para gestionar y proteger dichos activos.

¿Por qué es el eje central?

El DAMA Wheel posiciona a Data Governance en el centro porque sus principios y decisiones afectan a todos los demás Knowledge Areas:

Data Architecture Data Modeling & Design Data Storage & Operations Data Security Data Integration Documents & Content Reference & Master Data Data Warehousing & BI Metadata Management Data Quality
Drivers de Negocio

Las organizaciones adoptan DG para responder a necesidades concretas:

↳ Reducir riesgos: cumplimiento regulatorio, privacidad de datos, seguridad.
↳ Mejorar procesos: estandarización, calidad de datos, procesos de negocio más eficientes.
↳ Generar valor: mejor toma de decisiones, nuevas fuentes de ingreso, ventaja competitiva.
💡 Contexto clave: DG no es solo tecnología ni un proyecto de TI. Es un programa de cambio organizacional que requiere patrocinio ejecutivo, colaboración cross-funcional y gestión del cambio cultural.

Conceptos Esenciales

Antes de implementar Data Governance es necesario comprender sus componentes estructurales: la organización DG, el Data Stewardship, las políticas y la valoración de activos de datos.

1. Organización de Data Governance

Una organización DG típica tiene tres niveles de responsabilidad:

Estratégico
Data Governance Council
CDO / Executive Sponsors
Táctico
Data Governance Steering Committee
Data Stewardship Council
Operacional
Data Stewards
Data Owners / Analysts
2. Tipos de Data Stewards
Rol
Responsabilidad
Chief Data Steward
Preside cuerpos de DG; puede actuar como CDO en estructuras virtuales.
Executive Data Steward
Gerente senior que forma parte del Data Governance Council.
Enterprise Data Steward
Supervisión de un dominio de datos a través de múltiples funciones de negocio.
Business Data Steward
Experto en la materia; define y controla datos junto a stakeholders.
Technical Data Steward
Profesional TI (DBA, DQ Analyst, BI Specialist, Metadata Admin).
Coordinating Data Steward
Lidera equipos de stewards; clave en grandes organizaciones.
3. Data Policies vs. Standards vs. Procedures

La jerarquía normativa de DG sigue esta estructura:

Policies — "el QUÉ"
Directrices globales. Pocas, breves, directas. Codifican principios de gestión.
Standards — "el CÓMO (mínimo)"
Definen los requisitos mínimos de conformidad para implementar las políticas.
Procedures — "el CÓMO (detalle)"
Pasos operativos específicos para cumplir los estándares.
4. Valoración de Activos de Datos

El DMBOK2 propone traducir los Principios Contables Generalmente Aceptados (GAAP) a Generally Accepted Information Principles:

Asset Principle

Los datos son activos gestionables como otros activos materiales.

Liability Principle

Existe pasivo financiero por mal uso o mal manejo regulatorio de datos.

Quality Principle

La precisión y el ciclo de vida de los datos afectan el estado financiero.

Risk Principle

El riesgo asociado a los datos debe reconocerse formalmente.

Herramientas, Técnicas y Métricas

El capítulo describe herramientas concretas y un set de métricas para monitorear la efectividad del programa DG.

Herramientas Clave
Online Presence Portales web internos que comunican el programa DG: sus políticas, estándares, directrices y actividades del consejo.
Business Glossary Sistema de registro de términos de negocio. Los stewards son responsables de su creación y mantenimiento. Fundamental para la coherencia semántica.
Workflow Tools Automatizan aprobaciones de cambios en definiciones, políticas y estándares de datos. Mantienen la trazabilidad de decisiones.
DG Scorecards Tableros de control que muestran el cumplimiento de estándares y el progreso del programa DG frente a los KPIs establecidos.
Document Mgmt Repositorios centralizados para las políticas, procedimientos y documentación del programa DG.
